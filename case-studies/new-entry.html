Overview

A structured way to capture and manage multiple entries in a consistent format inside the Microsoft/SharePoint environment. Making the process easy for users while enforcing required fields, classification rules, and review readiness before submission.

The Problem

  • Users struggled with inconsistent prefix formatting.
  • Form fields lacked hierarchy and visual grouping.
  • Validation occurred only after submission, causing rework.
  • The UI felt outdated and did not match modern M365 patterns.

The Solution

I designed a clean, two‑panel interface with a left‑side creation form and a right‑side live preview. The interface guides users through required inputs, applies classification-based behavior to key fields, and prevents incomplete data from moving forward

Key Features
